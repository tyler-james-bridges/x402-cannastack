## Environment limitation (needs operator follow-up)

This verification ran in a sandboxed environment whose network egress allowlist
does not include `cannastack.0x402.sh` or `facilitator.x402.abs.xyz` (requests
return `403 x-deny-reason: host_not_allowed` from the egress proxy). The live
production check therefore could not be performed from here.

**To verify prod for real, run from an unrestricted machine:**

```bash
curl -i -X POST https://cannastack.0x402.sh/api/strain-finder \
  -H 'Content-Type: application/json' \
  -d '{"strain":"Blue Dream","location":"Phoenix, AZ"}'
# Expect: HTTP 402 + payment-required header (base64 JSON)
```

Also confirm in Vercel that `X402_PREVIEW_MODE` is **unset** in the production
environment — if it is set, all paid endpoints are served free.

## Local verification (production build, commit 0e7713a)

Built with `next build`, served with `next start`. Because the x402 resource
server fetches supported kinds from the facilitator at startup and the real
facilitator host is egress-blocked here, a local mock facilitator was used
(`GET /supported` → `{kinds:[{x402Version,scheme:"exact",network:"eip155:2741"}]}`,
all other routes erroring). The mock only affects facilitator discovery; the
402 challenge and payment requirements are generated entirely by app code, so
this verifies exactly what the deployed code serves.

### Unpaid POSTs → 402

All four endpoints returned `HTTP/1.1 402 Payment Required` with a
`payment-required` header. Decoded payloads (identical across endpoints except
resource URL/description):

```json
{
  "x402Version": 2,
  "scheme": "exact",
  "network": "eip155:2741",
  "amount": "20000",
  "asset": "0x84A71ccD554Cc1b02749b35d22F684CC8ec987e1",
  "payTo": "0x668aDd9213985E7Fd613Aec87767C892f4b9dF1c",
  "maxTimeoutSeconds": 300,
  "extra": { "name": "Bridged USDC (Stargate)", "version": "2", "decimals": 6 }
}
```

- `20000` units at 6 decimals = **$0.02 USDC** ✅
- Network `eip155:2741` = **Abstract** ✅
- Asset = Bridged USDC (Stargate) on Abstract ✅
- `payTo` = default wallet from `src/lib/x402.ts` (prod may override via
  `CANNASTACK_PAY_TO` — confirm the Vercel value is the intended wallet)

### Free surfaces → 200

`/llms.txt`, `/openapi.json`, `/.well-known/x402.json` all returned 200 with no
payment gate. The manifest's `payment` block matches the served requirements
(protocol x402 v2, exact scheme, `eip155:2741`, same USDC asset address,
facilitator `https://facilitator.x402.abs.xyz`), and every paid endpoint is
listed at `price_usdc: 0.02`.

### Preview-mode bypass

With `X402_PREVIEW_MODE=1`, an unpaid POST reached the handler (returned a 400
validation error rather than 402), confirming the bypass works and underscoring
that this var must stay unset in prod.

## Finding: facilitator outage causes 500s, not 402s

When the facilitator is unreachable, `x402ResourceServer.initialize()` throws
(`no supported payment kinds loaded from any facilitator`) and **unpaid
requests fail with HTTP 500 instead of receiving a 402 challenge**. Observed
directly during this run before the mock facilitator was in place.

Implication for prod: an outage of `facilitator.x402.abs.xyz` takes the paid
API fully down (500s) rather than degrading gracefully. Worth considering a
cached/fallback supported-kinds list or a clearer 503 + retry response.
